# vod2pod-rss - Project Context

## Project Overview

**vod2pod-rss** is a Rust-based web application that converts YouTube, Twitch, PeerTube, or generic RSS/Atom feed channels into podcast RSS feeds. The application:

- Converts video-on-demand (VoD) content into audio podcast feeds
- Transcodes videos to MP3 (or other audio formats) on-the-fly using ffmpeg
- Does not store downloaded content - streams directly to clients
- Uses Redis for caching generated feeds (default TTL: 600 seconds)
- Supports multiple audio codecs: MP3 (default), Opus, OGG Vorbis
- Works on ARM devices (Raspberry Pi 3-4)

### Architecture

The application consists of 4 main modules:

1. **`provider/`** - Implements the `MediaProvider` trait for different platforms:
   - `youtube.rs` - YouTube channel/playlist support
   - `twitch.rs` - Twitch channel support
   - `peertube.rs` - PeerTube instance support
   - `generic.rs` - Generic RSS/Atom feed and direct media URL support

2. **`rss_transcodizer/`** - Rewrites generated RSS feeds to inject transcoding URLs

3. **`transcoder/`** - Handles live ffmpeg transcoding with:
   - Range request support (seeking within streams)
   - Configurable bitrate (default: 192k)
   - Timeout handling

4. **`server/`** - Actix-web HTTP server exposing:
   - `/` - Web UI for generating feed URLs
   - `/transcodize_rss?url=<channel_url>` - Generate podcast RSS feed
   - `/transcode_media/to.mp3` - Stream transcoded audio
   - `/health` - Health check endpoint

### Key Dependencies

- **Web Framework**: Actix-web 4.12
- **Async Runtime**: Tokio 1.48
- **Caching**: Redis (via redis-rs 1.0 with cached 0.56)
- **HTTP Client**: reqwest 0.12
- **RSS Handling**: rss 2.0, feed-rs 2.3
- **Video Processing**: ffmpeg (external), yt-dlp (Python package)
- **YouTube API**: google-youtube3 5.0 (optional, for >15 results)

## Building and Running

### Prerequisites

- Rust toolchain (edition 2021)
- Docker & Docker Compose (for Redis)
- ffmpeg
- yt-dlp (`pip3 install yt-dlp`)
- Redis server

### Quick Start (Development)

```bash
# Install dependencies (Ubuntu/Debian)
make install-ubuntu-deps

# Or for Fedora
make install-fedora-deps

# Start Redis (required)
make start-deps

# Configure environment (edit .env with API keys if needed)
# .env file is auto-generated by install scripts

# Run tests
make test

# Run the server
make run

# Hot-reload on file changes
make hot-reload
```

### Docker Deployment

```bash
# Using docker-compose (production)
docker compose up -d

# Build custom image
make image
```

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `TRANSCODE` | `true` | Enable/disable transcoding |
| `MP3_BITRATE` | `192` | Audio bitrate in kbps |
| `SUBFOLDER` | `/` | Root path for reverse proxy support |
| `YT_API_KEY` | - | YouTube API key (optional, lifts 15-item limit) |
| `TWITCH_CLIENT_ID` | - | Twitch API client ID |
| `TWITCH_SECRET` | - | Twitch API secret |
| `REDIS_ADDRESS` | `localhost` | Redis server address |
| `REDIS_PORT` | `6379` | Redis server port |
| `CACHE_TTL` | `600` | Cache TTL in seconds |
| `VALID_URL_DOMAINS` | - | Comma-separated allowed domains |
| `AUDIO_CODEC` | `MP3` | Output codec (MP3/OPUS/OGG_VORBIS) |
| `YOUTUBE_YT_DLP_GET_URL_EXTRA_ARGS` | `[]` | JSON array of extra yt-dlp args |
| `FFMPEG_TIMEOUT_SECONDS` | `300` | ffmpeg operation timeout |

## Development Conventions

### Code Style

- Rust 2021 edition
- Uses `eyre::Result` for error handling throughout
- Async/await pattern with Tokio runtime
- Logging via `log` crate with `simple_logger`

### Testing

- Integration tests in `tests/integration_tests.rs`
- Unit tests alongside source files (`#[cfg(test)]` modules)
- Run tests: `cargo test -- --nocapture` or `make test`
- Tests require Redis running

### Provider Implementation

To add a new media provider:

1. Create a new file in `src/provider/` (e.g., `newplatform.rs`)
2. Implement the `MediaProvider` trait:
   ```rust
   #[async_trait]
   impl MediaProvider for NewPlatformProvider {
       async fn generate_rss_feed(&self, channel_url: Url) -> eyre::Result<String>;
       async fn get_stream_url(&self, media_url: &Url) -> eyre::Result<Url>;
       fn domain_whitelist_regexes(&self) -> Vec<Regex>;
   }
   ```
3. Register in `src/provider/mod.rs` via `generate_static_dispatcher!` macro

### Key Design Principles

1. **Provider Isolation**: Providers generate raw RSS feeds without knowledge of transcoding
2. **URL Whitelisting**: All URLs must match provider regex patterns (SSRF protection)
3. **Caching**: Feed generation results are cached in Redis
4. **Range Request Support**: Transcoder handles HTTP Range headers for seeking

## Common Issues

- **Redis connection failures**: Ensure Redis is running (`make start-deps`)
- **YouTube 15-item limit**: Set `YT_API_KEY` environment variable
- **Transcoding failures**: Verify ffmpeg and yt-dlp are installed and in PATH
- **SSRF protection errors**: Add domain to `VALID_URL_DOMAINS` or ensure provider regex matches
